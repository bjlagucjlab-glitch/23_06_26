# Online Store BI Analytics & Churn Scoring

Скрипт для аналитики данных интернет-магазина, проверки финансовых транзакций, расчета эффективности маркетинговой акции и скоринга риска оттока клиентов (Churn Rate).

Программа подключается к локальной базе данных SQLite (online_store.db), выгружает данные в датафреймы Pandas и решает 4 прикладные аналитические задачи.

## Стек технологий

* Python 3.8+
* Pandas, NumPy (ETL-процессы, очистка данных, расчет метрик)
* SQLite3 (SQL-запросы к БД проекта)
* Matplotlib, Seaborn (визуализация данных)

## Структура проекта

Для корректной работы программы файл базы данных должен находиться в корневой директории:
```text
├── main.py               # Основной исполняемый скрипт
├── online_store.db       # База данных SQLite (должна лежать рядом со скриптом)
└── README.md
```

При запуске скрипт проверяет наличие online_store.db. Если файл не найден, выполнение программы прекратится с ошибкой SystemExit.
## Описание логики работы

Код разделен на 6 функциональных блоков:

### 1. Сбор данных (ETL)

Написано 11 изолированных SQL-запросов, выгружающих таблицы в DataFrame:

* customers (включая PII-данные)
* products, categories, suppliers
* orders, order_items, returns
* inventory, shipments, reviews, payments, promotions

### 2. Предобработка данных

* Заполнение пропусков в столбце email значением 'no_data'.
* Приведение дат (signup_date, birth_date) к типу datetime64.
* Нормализация названий стран: текстовые вариации приводятся к единому ISO-коду (например, Poland, Polska, pl -> PL; United States, USA -> US).

### 3. Анализ продуктового рейтинга (Задача 2.6)

* Рассчитывается динамика изменения среднего рейтинга товаров за 2022-2026 годы.
* Проводится корреляционный анализ зависимости между количеством отзывов и рейтингом товара.
* Сравниваются средние оценки товаров в разрезе клиентских сегментов.
* В коде заложены шаблоны графиков: Lineplot по месяцам, Boxplot по категориям и Scatterplot (временно отключены через комментирование).

### 4. Финансовый контроль

* Сверка таблиц orders и payments. Скрипт ищет выполненные заказы (status = 'completed'), у которых отсутствует запись об оплате.

### 5. Расчет акции "День рождения"

* Демографический анализ: распределение клиентов по месяцам рождения и проверка аудитории на совершеннолетие (18+).
* Проверка гипотезы о сезонности: сравнение активности клиентов, родившихся в разные времена года.
* Расчет юнит-экономики: оценка потенциальной выручки от предоставления скидки 10% именинникам текущего месяца.
* Визуализация распределения именинников через Pie Chart.

### 6. Скоринг риска оттока (Задача 2.2)

Для каждого клиента формируется вектор признаков:

1. Доля возвратов (return_rate) как отношение returned_orders к total_orders.
2. Средняя оценка клиента в отзывах (avg_rating).
3. Динамика снижения частоты покупок в днях (frequency_decline).
4. Количество дней с момента последней покупки (Recency).

Правила сегментации риска:

* Low Risk: последняя покупка была 90 или менее дней назад.
* Medium Risk: последняя покупка была от 91 до 180 дней назад.
* High Risk: с момента последней покупки прошло более 180 дней.

Результат выводится в виде Stacked Bar Chart, отображающего доли групп риска внутри каждого клиентского сегмента.

## Запуск проекта

1. Склонируйте репозиторий:

```bash
git clone [https://github.com/username/repository-name.git](https://github.com/username/repository-name.git)
cd repository-name

```
2. Установите зависимости:

```bash
pip install numpy pandas matplotlib seaborn

```
3. Убедитесь, что файл online_store.db находится в одной папке с main.py.
4. Запустите скрипт:

```bash
python main.py

```
